# 📁 프로젝트 파일 정리 가이드

## ✅ 필수 파일 (애플리케이션 실행에 반드시 필요)

### 핵심 파일
- **index.html** - 메인 HTML 파일 (필수)
- **styles.css** - 스타일시트 (필수)
- **app.js** - 메인 JavaScript 로직 (필수)
- **data.js** - 가게 데이터 (필수)
- **config.js** - API 키 설정 (필수, 직접 생성 필요)
- **config.js.example** - 설정 예시 파일 (유용, GitHub에 포함)

### Git 설정
- **.gitignore** - Git 제외 파일 목록 (필수)

---

## 🔧 유용한 파일 (개발 편의)

### 서버 실행 스크립트
- **start-server.bat** - Windows 배치 파일로 서버 실행 (유용)
- **start-server.ps1** - PowerShell 스크립트로 서버 실행 (유용)
  - ⚠️ 둘 중 하나만 있어도 됨 (Windows 사용자라면 .bat 권장)

### Git 작업 스크립트
- **git-push.bat** - Git 푸시 자동화 (선택적)
- **git-setup.bat** - Git 초기 설정 (선택적)
  - ⚠️ Cursor나 다른 Git 도구를 사용한다면 불필요할 수 있음

---

## ❌ 불필요하거나 정리 가능한 파일

### 중복 문서 (통합 권장)
1. **GitHub연결단계별가이드.md** → `docs/Git설정가이드.md`에 통합
2. **docs/API인증오류해결.md** → `docs/문제해결가이드.md`에 통합
3. **docs/Web서비스URL등록가이드.md** → `docs/문제해결가이드.md`에 통합
4. **docs/UTMK좌표변환가이드.md** → `docs/가게추가가이드.md`에 통합
5. **docs/좌표찾는방법.md** → `docs/가게추가가이드.md`에 통합
6. **docs/길찾기기능가이드.md** → `docs/실행방법.md`에 통합
7. **docs/README.md** → 루트 `README.md`와 중복, 삭제 가능

### 선택적 파일 (사용 안 하면 삭제 가능)
- **git-push.bat** - Cursor나 다른 Git 도구 사용 시 불필요
- **git-setup.bat** - Cursor나 다른 Git 도구 사용 시 불필요
- **start-server.ps1** - `.bat` 파일 사용 시 불필요 (또는 반대)

## 📊 정리 권장 사항

### 즉시 정리 가능
1. **docs/README.md** 삭제 (루트 README.md와 중복)
2. **GitHub연결단계별가이드.md** → `docs/` 폴더로 이동 또는 통합

### 통합 권장
1. Git 관련 문서들을 하나로 통합:
   - `docs/Git설정가이드.md`
   - `docs/GitHub푸시가이드.md`
   - `docs/Cursor에서Git연결하기.md`
   - → `docs/Git가이드.md` 하나로 통합

2. 문제 해결 문서 통합:
   - `docs/API인증오류해결.md`
   - `docs/Web서비스URL등록가이드.md`
   - → `docs/문제해결가이드.md`에 통합

3. 가게 추가 관련 통합:
   - `docs/UTMK좌표변환가이드.md`
   - `docs/좌표찾는방법.md`
   - → `docs/가게추가가이드.md`에 통합

## 💡 정리 팁

1. **문서는 필요할 때만 참고**: 사용하지 않는 문서는 삭제해도 됩니다
2. **Git 스크립트**: Cursor나 다른 Git 도구를 사용한다면 `.bat` 파일 불필요
3. **서버 스크립트**: `.bat`와 `.ps1` 중 하나만 유지
4. **문서 통합**: 비슷한 내용의 문서는 하나로 통합하는 것이 관리하기 쉬움
