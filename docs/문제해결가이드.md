# 🔧 네이버 지도 API 문제 해결 가이드

## 1단계: 브라우저 콘솔에서 오류 확인

### 방법
1. 브라우저에서 `index.html` 열기
2. **F12** 키 누르기 (개발자 도구 열기)
3. **Console** 탭 클릭
4. 빨간색 오류 메시지 확인

### 주요 오류 메시지

#### "Invalid client id" 또는 "클라이언트 ID가 유효하지 않습니다"
- **원인**: API 키가 잘못되었거나 등록되지 않음
- **해결**: 네이버 클라우드 플랫폼에서 Client ID 확인

#### "This domain is not allowed" 또는 "허용되지 않은 도메인"
- **원인**: Web 서비스 URL이 등록되지 않음
- **해결**: 네이버 클라우드 플랫폼에서 Web 서비스 URL 등록

#### "naver is not defined"
- **원인**: 네이버 지도 API 스크립트가 로드되지 않음
- **해결**: API 키 확인 및 네트워크 연결 확인

---

## 2단계: 네이버 클라우드 플랫폼 설정 확인

### 확인 사항

1. **Application이 활성화되어 있는지**
   - 콘솔 → AI·NAVER API → Application
   - 상태가 "활성"인지 확인

2. **Web 서비스 URL이 등록되어 있는지**
   - Application 상세 페이지로 이동
   - "서비스 환경" 섹션 확인
   - 다음 중 하나가 등록되어 있어야 함:
     - `http://localhost`
     - `http://localhost:8000`
     - `http://127.0.0.1:8000`
     - 또는 파일을 직접 열 경우: `file://`

3. **Client ID 복사**
   - Application 상세 페이지에서 Client ID 확인
   - 정확히 복사했는지 확인 (공백 없이)

---

## 3단계: 코드 확인

### index.html 72번째 줄 확인

올바른 형식:
```html
<script type="text/javascript" src="https://oapi.map.naver.com/openapi/v3/maps.js?ncpClientId=여기에_실제_Client_ID"></script>
```

잘못된 예:
- `ncpClientId=YOUR_CLIENT_ID` (변경 안 함)
- `ncpClientId= 9ajf9p6dfh` (공백 있음)
- `ncpClientId="9ajf9p6dfh"` (따옴표 있음)

---

## 4단계: 파일 직접 열기 시 특별 설정

파일을 직접 열 경우 (`file://` 프로토콜), 네이버 클라우드 플랫폼에서:
- Web 서비스 URL에 `file://` 추가는 불가능
- **해결책**: 로컬 서버 사용 또는 `http://localhost` 등록 후 서버 실행

---

## 5단계: 캐시 삭제

브라우저가 이전 버전을 캐시했을 수 있습니다:

1. **강력 새로고침**
   - `Ctrl + Shift + R` (Windows)
   - `Ctrl + F5`

2. **브라우저 캐시 삭제**
   - 설정 → 인터넷 사용 기록 삭제
   - 캐시된 이미지 및 파일 선택 후 삭제
