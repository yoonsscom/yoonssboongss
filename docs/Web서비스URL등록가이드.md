# 🔧 Web 서비스 URL 등록 상세 가이드

## 현재 상황
- **URI**: `http://127.0.0.1:5500/index.html`
- **오류**: 인증 실패 (Web 서비스 URL 미등록)

## 정확한 등록 방법

### 중요: URL 형식 확인

네이버 클라우드 플랫폼에서 Web 서비스 URL을 등록할 때:

#### ✅ 올바른 형식
- `http://127.0.0.1:5500` (포트 번호 포함, 경로 제외)
- `http://localhost:5500` (포트 번호 포함, 경로 제외)

#### ❌ 잘못된 형식
- `http://127.0.0.1:5500/index.html` (경로 포함하면 안 됨)
- `127.0.0.1:5500` (프로토콜 없음)
- `http://127.0.0.1` (포트 번호 없음 - 작동할 수도 있지만 포트 포함 권장)

## 단계별 등록 방법

### 1단계: 네이버 클라우드 플랫폼 접속
1. https://www.ncloud.com/ 접속
2. 로그인

### 2단계: Application 찾기
1. 콘솔 → **AI·NAVER API** → **Application**
2. Application 목록에서 등록한 Application 찾기
3. Application 이름 클릭 (상세 페이지로 이동)

### 3단계: Application 수정
1. 상단 또는 우측의 **"수정"** 버튼 클릭

### 4단계: Web 서비스 URL 등록
1. 페이지를 아래로 스크롤하여 **"서비스 환경 등록"** 섹션 찾기
2. **"Web 서비스 URL"** 필드 찾기
3. 다음 URL 입력:
   ```
   http://127.0.0.1:5500
   ```
   ⚠️ **주의**: 
   - `/index.html` 같은 경로는 **포함하지 않습니다**
   - 포트 번호(`:5500`)는 **반드시 포함**해야 합니다
   - `http://` 프로토콜도 **반드시 포함**해야 합니다

4. **"+ 추가"** 버튼 클릭 (필요한 경우)
5. 입력한 URL이 목록에 표시되는지 확인

### 5단계: 추가 URL 등록 (권장)
개발 편의를 위해 여러 URL을 등록하는 것을 권장합니다:

1. `http://127.0.0.1:5500`
2. `http://localhost:5500`
3. `http://127.0.0.1:8000` (다른 포트 사용 시)
4. `http://localhost:8000`

### 6단계: 저장
1. 페이지 하단의 **"수정"** 또는 **"저장"** 버튼 클릭
2. 저장 완료 메시지 확인

### 7단계: 브라우저 새로고침
1. 브라우저에서 **강력 새로고침**
   - `Ctrl + Shift + R` (Windows)
   - 또는 `Ctrl + F5`
2. 콘솔(F12)에서 오류 메시지 확인

## 확인 체크리스트

등록 후 확인:
- [ ] Application 상세 페이지에서 "서비스 환경" 섹션에 URL이 표시되는가?
- [ ] URL 형식이 정확한가? (`http://127.0.0.1:5500`)
- [ ] 저장 버튼을 클릭했는가?
- [ ] 브라우저를 강력 새로고침했는가? (`Ctrl + Shift + R`)

## 여전히 안 되나요?

### 확인 사항
1. **Application 상태**: "활성" 상태인지 확인
2. **Client ID**: 정확한 Client ID인지 확인 (공백 없이)
3. **URL 형식**: 정확한 형식인지 다시 확인
4. **저장**: 수정 후 반드시 저장 버튼 클릭했는지 확인
5. **캐시**: 브라우저 캐시 삭제 후 다시 시도

### 대안
만약 계속 안 된다면:
- 다른 포트 번호 사용 (예: 8000)
- `http://localhost` 대신 `http://127.0.0.1` 사용
- 네이버 클라우드 플랫폼 고객 지원 문의

## 참고
- URL 등록 후 즉시 반영되지만, 때로는 몇 분 걸릴 수 있습니다
- 여러 URL을 등록하면 다양한 환경에서 테스트할 수 있습니다
- 포트 번호가 바뀌면 새로운 URL을 추가로 등록해야 합니다
